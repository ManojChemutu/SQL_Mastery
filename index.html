🧠
API Server
kube-apiserver
Single point of truth. All components communicate via REST. Handles auth, RBAC, admission control, and persists state to etcd.
  • Authentication (x509, OIDC, SA tokens)
  • Authorization (RBAC, ABAC, Webhook)
  • Admission Controllers (Mutating/Validating Webhook)
  • Watch-based event streaming
🗄
etcd
Distributed KV Store
Stores all cluster state. Distributed consensus via Raft protocol. Must be backed up regularly — this IS the cluster.
  • Raft consensus — odd node count (3 or 5)
  • Backup: etcdctl snapshot save
  • Encrypt secrets at rest (EncryptionConfiguration)
  • Watch API for change notifications
📅
Scheduler
kube-scheduler
Watches for unscheduled Pods and assigns them to the best-fit Node using filtering, scoring, and custom plugins.
  • Filtering → Scoring pipeline
  • NodeAffinity / PodAffinity / Anti-Affinity
  • Taints & Tolerations
  • Custom scheduler framework plugins
🔄
Controller Manager
Reconciliation loops
Watches desired vs actual state and takes corrective action. ReplicaSet, Deployment, Job controllers all live here.
  • ReplicaSet controller (replica count)
  • Deployment controller (rolling updates)
  • Node controller (heartbeat)
  • CronJob controller
🤝
kubelet
Node Agent
Runs on every worker node. Communicates with containerd to start/stop containers. Reports status back to API server.
  • PodSpec execution via CRI
  • Volume mounting (CSI)
  • Probe execution (liveness/readiness)
  • Node resource reporting
🌐
kube-proxy
Network Rules
Maintains iptables/ipvs rules for Service-to-Pod routing. Implements ClusterIP virtual IPs and load-balancing.
  • iptables mode (default)
  • IPVS mode (high perf — BFSI recommended)
  • eBPF mode (Cilium replacement)
  • EndpointSlices for large clusters
kubectl apply → What Happens
request lifecycle
1.  kubectl → kubeconfig → API Server (:6443) // TLS + Auth
2.  Auth → RBAC check → Admission Controllers
3.  Persisted to etcd
4.  Deployment Controller watches → creates ReplicaSet
5.  ReplicaSet Controller → creates N Pods (Pending)
6.  Scheduler → assigns Node (Scheduled)
7.  kubelet → calls containerd → pulls image (Running)
8.  kubelet → runs probes → reports Ready
9.  kube-proxy → updates iptables rules

Workload Objects
Pods, Deployments, StatefulSets, DaemonSets, Jobs — when to use what.
📦
Pod
Atomic unit
Smallest deployable unit. One or more containers sharing network namespace and volumes. Rarely created directly — use controllers.
pod.yaml
spec:
  containers:
  - name: api
    image: payment-api:2.1.0
    resources:
      requests: {cpu: 100m, memory: 256Mi}
      limits:   {cpu: 500m, memory: 512Mi}
    livenessProbe:
      httpGet:
        path: /health
        port: 8080
🚀
Deployment
Stateless workloads
Manages ReplicaSets. Rolling updates, rollbacks, desired replica count. Go-to for stateless microservices.
rolling update strategy
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxSurge: 1       # extra pods
    maxUnavailable: 0 # zero downtime
minReadySeconds: 30

# Rollback
kubectl rollout undo deploy/payment-api
🗂
StatefulSet
Stateful workloads
Stable pod identity, ordered start/stop, stable persistent storage. Use for Kafka, Zookeeper, Oracle DB.
  • Stable network identity: pod-0, pod-1, pod-2
  • volumeClaimTemplates — PVC per pod
  • Ordered rolling updates
  • Headless service for direct pod DNS
  • podManagementPolicy: Parallel
👁
DaemonSet
Node-level agents
Ensures one pod per node. Perfect for log collectors, monitoring agents, network plugins.
  • Fluentd / Filebeat log collection
  • node-exporter (Prometheus metrics)
  • Falco security runtime agent
  • CNI plugins (Calico, Cilium)
⏰
Jobs & CronJobs
Batch workloads
cronjob.yaml
schedule: "0 2 * * *"   # 2am daily
concurrencyPolicy: Forbid
successfulJobsHistoryLimit: 3
jobTemplate:
  spec:
    backoffLimit: 3
    activeDeadlineSeconds: 3600
🔁
HPA / VPA / KEDA
Autoscaling
  • HPA: scale replicas on CPU/memory %
  • VPA: adjust resource requests automatically
  • KEDA: scale on Kafka lag, queue depth
  • KEDA: ScaleToZero for cost optimization
  • PDB: minAvailable during disruptions
Pod Design Patterns
🚗
Sidecar
Extends main container. Envoy proxy, log shipper, secret injector. Shares network and volumes within same Pod.
🚪
Init Container
Runs before app containers. DB migration, config fetch, wait-for-dependency. Must exit 0 to proceed.
📡
Ambassador
Proxy sidecar handling TLS, retries, circuit breaking between main container and external services.
🌐
Networking
Services, Ingress, Network Policies, DNS, Service Mesh.
K8s Networking Model
Every pod gets a unique IP. Pods communicate directly without NAT. Implemented by CNI plugins (Calico, Cilium, Flannel). Services provide stable virtual IPs over dynamic pods.
🔌
Service Types
Stable endpoints
  • ClusterIP — internal only (default)
  • NodePort — node :30000-32767
  • LoadBalancer — cloud LB (Azure/OCI)
  • ExternalName — CNAME alias
  • Headless (clusterIP: None) — StatefulSet DNS
service.yaml
spec:
  selector: {app: payment-api}
  ports:
  - port: 80
    targetPort: 8080
  sessionAffinity: ClientIP
🚦
Ingress
L7 HTTP routing
ingress.yaml
rules:
- host: api.citi.internal
  http:
    paths:
    - path: /payments
      backend:
        service:
          name: payment-api
          port: {number: 80}
tls:
- secretName: tls-cert
  hosts: [api.citi.internal]
🛡
Network Policies
BFSI critical
default-deny + allow
# Deny all ingress/egress
spec:
  podSelector: {}
  policyTypes: [Ingress,Egress]

# Allow only api-gateway ns
ingress:
- from:
  - namespaceSelector:
      matchLabels:
        name: api-gateway
🔗
Service Mesh (Istio)
mTLS + traffic mgmt
  • Automatic mTLS between all services
  • VirtualService — traffic splitting
  • DestinationRule — circuit breaking
  • Canary: 90/10 traffic split
  • AuthorizationPolicy — service RBAC
  • Jaeger integration — distributed tracing
CoreDNS Naming
cluster DNS format
# Full format:
<service>.<namespace>.svc.cluster.local

# Examples:
payment-api.production.svc.cluster.local
kafka.messaging.svc.cluster.local

# StatefulSet pod DNS (headless):
kafka-0.kafka-headless.messaging.svc.cluster.local

# From same namespace — shorthand works:
curl http://payment-api/health

Storage & Persistence
Volumes, PVs, PVCs, StorageClasses and CSI drivers for stateful workloads.
💿
PV Lifecycle
  • StorageClass → dynamic provisioning
  • PVC → binds to PV (1:1)
  • Pod mounts PVC as volume
  • ReclaimPolicy: Retain | Delete
  • AccessModes: RWO | ROX | RWX | RWOP
pvc.yaml
spec:
  accessModes: [ReadWriteOnce]
  storageClassName: azure-managed
  resources:
    requests:
      storage: 50Gi
🗃
StorageClass Examples
Azure / OCI / On-prem
  • azure-managed-premium (SSD, RWO)
  • azure-file (NFS-based, RWX)
  • oci-block-storage (Block, RWO)
  • nfs-provisioner (on-prem, RWX)
  • local-storage (bare metal, fastest)
🔧
Volume Types
  • emptyDir — ephemeral, pod lifetime
  • hostPath — node filesystem (caution)
  • configMap / secret — config injection
  • projected — combine multiple sources
  • CSI — external storage plugins
🏦
DB on K8s — BFSI
  • StatefulSet for Oracle DB (OCI CSI)
  • Velero for PV backup to object storage
  • Separate namespace + NetworkPolicy
  • Anti-affinity across AZs
  • ResourceQuota to isolate noisy neighbors

Security
RBAC, PodSecurity, Secrets management, OPA, runtime security. Critical for BFSI compliance.
BFSI Security Baseline
Default-deny NetworkPolicies · No privileged containers · Secrets encrypted at rest · Image scanning in CI · Audit logging to SIEM · CyberArk/Vault for dynamic secrets · Least-privilege RBAC per ServiceAccount.
👥
RBAC
Role-Based Access Control
least-privilege role
kind: Role
rules:
- apiGroups: [""]
  resources: [pods,pods/log]
  verbs: [get,list,watch]
# Never cluster-admin for app SAs
🔒
Pod Security
PSA (K8s 1.25+)
  • privileged — no restrictions (avoid)
  • baseline — prevents obvious escapes
  • restricted — hardened (BFSI target)
  • runAsNonRoot: true
  • readOnlyRootFilesystem: true
  • allowPrivilegeEscalation: false
  • seccompProfile: RuntimeDefault
🗝
Secrets Management
  • Encrypt etcd at rest (EncryptionConfig)
  • External Secrets Operator → CyberArk/Vault
  • Vault Agent Injector (sidecar)
  • Never bake secrets into images
  • Rotate without pod restart (projected vol)
📋
OPA / Gatekeeper
Policy as Code
rego — require limits
violation[{"msg": msg}] {
  c := input.review.object
         .spec.containers[_]
  not c.resources.limits
  msg := "Limits required"
}
👁
Runtime Security
Falco + Audit Logs
  • Falco: detects syscall anomalies
  • Alert: shell in container, sensitive file read
  • K8s Audit Policy → Splunk / SIEM
  • Image scanning: Trivy in CI, Twistlock prod
  • Only signed images (Cosign/Notary)
🧱
OpenShift SCC
SecurityContextConstraints
  • OCP alternative to PodSecurityAdmission
  • restricted-v2 (default / BFSI baseline)
  • anyuid — legacy apps needing root
  • privileged — infra components only
  • Assigned via RoleBinding to ServiceAccount

SLO Definition
SLO — payment API availability
# Error rate SLI
sum(rate(http_requests_total{job="payment-api",code=~"5.."}[5m]))
/
sum(rate(http_requests_total{job="payment-api"}[5m]))

# SLO: 99.9% = error_rate < 0.001
# Error budget: 43.8 min/month

# Burn rate alert
alert: SLOBurnRateHigh
expr: error_rate > 0.05
for: 5m
